Fawn-ftp

知识点

  • ftp - 21/tcp
  • 文件传输协议(FTP)是一种标准的通信协议，用于在计算机网络上将计算机文件从服务器传输到客户端。
  • FTP建立在客户机-服务器模型体系结构上，在客户机和服务器之间使用单独的控制和数据连接。FTP用户可以使用明文登录协议进行身份验证，通常采用用户名和密码的形式。但是，如果服务器配置允许，它们可以匿名连接。
  • 为了保证用户名和密码的安全传输，并对内容进行加密，FTP通常使用FTPS (SSL/TLS)保护，或者使用SFTP (SSH File Transfer Protocol)代替。

Dancing-smb

知识点

  • smb - 445/tcp
  • 使用SMB协议，应用程序(或应用程序的用户)可以访问远程服务器上的文件以及打印机等其他资源。
  • 网络中启用smb的存储称为共享。拥有服务器地址和适当凭证的任何客户机都可以访问这些文件。
  • 尽管能够保护对共享的访问，但是网络管理员有时会犯错误，意外地允许在没有任何有效凭据的情况下登录，或者使用guest accounts或anonymous log-ons。
  • smb一些共享名称的意思：
    • ADMIN$—管理共享是由Windows NT系列操作系统创建的隐藏网络共享，它允许系统管理员远程访问网络连接系统上的每个磁盘卷。这些共享可能不会被永久删除，但可能会被禁用。
    • C$ - C:\磁盘卷的管理共享。这是承载操作系统的位置。
    • IPC$——进程间通信共享。用于通过命名管道进行进程间通信，不是文件系统的一部分。
    • WorkShares -自定义共享。

Redeemer-redis

知识点

  • redis（REmote DIctionary Server） - 6379/tcp

  • redis内存数据库，主要依赖于主内存进行数据存储（数据库在系统的RAM中进行管理），通常用于缓存频繁请求的数据，以便快速检索。

  • redis-cli比netcat更方便连接redis
